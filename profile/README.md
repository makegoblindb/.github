## Hi there 👋

O **GoblinDB** é uma organização focada no desenvolvimento de infraestrutura de dados moderna, priorizando a eficiência binária sem perder a flexibilidade dos bancos de dados orientados a documentos.

### 👺 Sobre o GoblinDB
Nossa organização dedica-se à criação de um ecossistema de banco de dados robusto, baseado na especificação **BMD (Binary Markdown)**. Acreditamos que a interoperabilidade nasce de documentações canônicas rigorosas e de uma arquitetura em camadas bem definida.

### 🌈 Como contribuir
A comunidade é o coração do projeto! Você pode se envolver das seguintes formas:
* **Especificação:** Revisando e propondo melhorias em nossos protocolos e formatos em `/docs`.
* **Código:** Implementando melhorias no core em Go ou desenvolvendo drivers em outras linguagens.
* **Testes:** Criando novos fixtures e amostras de dados (samples) para garantir a conformidade do protocolo.

### 👩‍💻 Recursos Úteis
* **Documentação Canônica:** Tudo sobre o formato BMD e o protocolo GBDB está no repositório principal.
* **Especificação de Arquivo:** Veja o `FILE-FORMAT-SPEC.md` para entender o layout on-disk.
* **Guia de Arquitetura:** Explore nossa estrutura de pastas seguindo o padrão Go.

### 🍿 Fun Facts
* Nosso formato de arquivo (BMD) é o que acontece quando o Markdown e um editor hexadecimal se apaixonam.
* Embora o nome seja "Goblin", nosso código é organizado o suficiente para deixar qualquer mago orgulhoso.
* Nossa meta é que o handshake do nosso protocolo seja mais rápido que um café expresso.
